How HTTP Began

1991: HTTP/0.9 – Just Text

Tim Berners-Lee created HTTP at CERN to help scientists share research papers. The first version supported only the GET command and returned plain text—no images, headers, or error codes.

Despite its simplicity, it introduced the client-server model still used today and established the principle of open, universal standards that fueled the web’s rapid academic adoption.

1996: HTTP/1.0 – A Richer Web

As the web grew beyond academia, users demanded multimedia and interactivity. HTTP/1.0 responded with:

  • Standard status codes like 200 OK and 404 Not Found
  • Headers to describe content type (e.g., images, HTML)
  • New methods like POST for form submissions

However, each file required a new TCP connection—making complex pages slow to load.

These enhancements enabled multimedia content and early e-commerce, laying the groundwork for the interactive, commercial web of the late 1990s.

1997: HTTP/1.1 – The Workhorse

HTTP/1.1 solved performance bottlenecks with:

  • Persistent connections – reuse one connection for multiple files
  • Pipelining – send several requests without waiting
  • The Host header – enabling multiple websites on one server

It became the foundation of the commercial web for nearly two decades.
