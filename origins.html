How HTTP Began

1991: HTTP/0.9 – Just Text

Tim Berners-Lee created HTTP at CERN so scientists could share research papers. The first version only supported the GET command and returned plain text—no images, no headers, no error codes. But it introduced the client-server model still used today.

1996: HTTP/1.0 – A Richer Web

As the web grew, users wanted images, forms, and dynamic content. HTTP/1.0 added:

  • Standard status codes like 200 OK and 404 Not Found
  • Headers to describe content type (e.g., images, HTML)
  • New methods like POST for form submissions
But each file (image, script, etc.) needed a new connection—making pages slow to load.

1997: HTTP/1.1 – The Workhorse

HTTP/1.1 fixed performance with:

  • Persistent connections – reuse one connection for multiple files
  • Pipelining – send several requests without waiting
  • The Host header – allowing multiple websites on one server
It powered the web for nearly 20 years.
